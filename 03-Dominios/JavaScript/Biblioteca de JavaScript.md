# Biblioteca de JavaScript

> Não há um Real Python único de JavaScript — o ecossistema é fragmentado demais (linguagem, runtime, frameworks, ferramentas). A referência se monta com 3–4 fontes complementares.

## Linguagem

- **[MDN Web Docs](https://developer.mozilla.org)** — documentação oficial-comunitária — A Bíblia. Não é tutorial-style como Real Python, mas referência de qualidade absurda. Os guias (não só a reference) são muito bem escritos.
- **[JavaScript.info](https://javascript.info)** — livro online — O que mais lembra Real Python em formato: progressivo, didático, profundo, gratuito. Provavelmente o melhor recurso para dominar a linguagem em si.
- **[2ality](https://2ality.com)** (Axel Rauschmayer) — blog pessoal — Deep dives em features do ES. Para quem quer entender *por que* algo funciona de tal jeito.

## Plataforma web e performance

- **[web.dev](https://web.dev)** — portal (Google) — Performance, Core Web Vitals, APIs modernas do browser. Estilo "case study + best practice".

## React e ecossistema de UI

- **[Josh Comeau](https://www.joshwcomeau.com/)** — blog pessoal — Para React/CSS, conteúdo dos mais bem produzidos que existem (interativo, visual, profundo).
- **[Kent C. Dodds](https://kentcdodds.com/blog)** — blog pessoal — React, testing, padrões. Opinativo mas substancial.
- **[Overreacted](https://overreacted.io)** (Dan Abramov) — blog pessoal — Para entender React por baixo, não há substituto.
- **[react.dev](https://react.dev)** — documentação oficial — Os novos docs do React substituem qualquer tutorial introdutório.

## Node.js

- **[Node.js docs](https://nodejs.org/docs)** — documentação oficial
- **[NodeSource blog](https://nodesource.com/blog)** — blog corporativo — Posts ocasionais de qualidade sobre runtime.
- **[NearForm blog](https://www.nearform.com/blog)** (Matteo Collina) — blog corporativo — Autor do Fastify e Pino, escreve sobre performance e arquitetura Node.
