# Validação no Frontend

Validar dados no cliente melhora a experiência do usuário com feedback imediato e reduz requisições inválidas ao servidor. A escolha da biblioteca impacta bundle size, integração com TypeScript e ergonomia de uso.

## Alternativas

- [[03-Domínios/Frontend/Validação/Zod|Zod]] — validação com inferência de tipos TypeScript, padrão atual no ecossistema
- [[03-Domínios/Frontend/Validação/Yup|Yup]] — schema-based, popular com Formik e React Hook Form
- [[03-Domínios/Frontend/Validação/Joi|Joi]] — robusto e maduro, mais comum no Node.js mas utilizável no frontend
